# codex-team 0.4.0

A restructured prompt layer (skills / commands) plus a history-display rewrite so Claude gets a cleaner, chronological view of what each Codex worker did. No daemon-protocol changes; existing installs upgrade in place.

## Highlights

- **New `codex-team-playbooks` skill** — a library of 9 multi-agent collaboration patterns plus an anti-patterns catalogue. Each playbook is a decision-tree-ready recipe: team composition, shared artefacts, communication protocol, iteration loop, copy-pasteable send templates, exit criteria, failure modes.
- **Skill layout consolidated** from 7 skills to 5. Details moved into on-demand reference files under each skill so `SKILL.md` stays thin.
- **`history.md` renders as a chronological Timeline** instead of grouped File changes / Commands / Messages / Final answer sections. Agent messages are fenced as markdown; multi-line commands as shell; file changes carry the correct `A / M / D / R` kind.

### `philosophy.md` is now the single source of truth

The 8 collaboration principles live in one place. Other skills link to section numbers (`philosophy.md §N`) rather than restating.

### Auto-load banner in every `SKILL.md`

Each skill opens with a short reminder telling Claude *why* it was auto-loaded — so if the trigger doesn't match, Claude knows to close the file and continue.

### Slash commands slimmed to decision trees

`bootstrap`, `shutdown`, `brief`, `heal`, `watch`, `workspaces`, `tutorial` — the procedures compressed, the judgment delegated to the skills. Slash command names are unchanged.

### History display rewrite

- **Chronological Timeline.** Items render in the order Codex emitted them. No more File changes → Commands → Messages → Final answer grouping.
- **Agent messages fenced.** Each `msg` body goes into a ```` ```markdown ```` block, full content, no truncation, no bullet-list mangling.
- **Multi-line commands fenced.** Multi-line shell commands go into ```` ```sh ```` blocks. Single-line commands render inline as code spans.
- **Correct file-change kind.** Added files report `[file A]`, modified `[file M]`, deleted `[file D]`, renamed `[file R]`. Line counts inferred from `content` when not explicit.
- **Final answer inline.** The final-answer message is labelled `**msg (final):**` in its chronological position. The separate "Final answer" section is gone (no more duplication).
- **Dynamic fence delimiters.** When an agent message or shell command contains its own fenced block, the outer fence uses more backticks so nesting is always safe.
- **Usage line.** A short `### Usage` block reports `tokens_last · tokens_total · files=+N/-M` up top; errors surface there too.

## Deprecated

- `[digest].agent_message_full` — agent messages are always full-text in 0.4.0. Flag retained in `config.toml` for forward compat but ignored.
- `[digest].max_files_listed` — the new Timeline emits every file change. Flag retained but ignored.

## Migration

**None required.** Existing `history.md` files remain readable (the section-splitting regex targets only the `## Turn …` header). New turns use the new Timeline format; mixed files render correctly.

## Known issues

Five daemon-server integration tests in `test/server.test.ts` fail on 0.4.0 as they did on 0.3.0 — unrelated to the history or skill refactor. They will be addressed in a follow-up. The digest, history, and prompt-layer changes have full passing coverage (12 new tests, all green).

## For orchestrators — what you'll notice

- `Monitor` events look the same. `turn-done` payload schema is unchanged.
- `codex-team history <name> --format md` output shape changes. If you were grepping `### File changes` / `### Commands` / `### Messages`, grep `### Timeline` or the `[cmd …]` / `[file …]` / `msg:` anchors instead.
- Every Codex turn now feels like reading a conversation instead of a report.
