# codex-team 0.5.3

Historical note: the examples below reflect the 0.5.3-era success envelope. Current mainline emits concise bare JSONL for successful non-streaming commands and pretty-prints `--full`.

The **UX + observability + bounded-memory** release. Motivated by the R1/R2/R3 code review and two rounds of dogfood testing. 0.5.2 shipped compact event digests and the `--short` plain-text mode; 0.5.3 ships the answer to "why is my daemon hanging silently?" plus the missing cohort, lifecycle, and health commands.

## Output: concise by default, `--full` for everything

**Breaking change.** The default JSON response body is now the *concise projection* — only the fields Claude actually needs to act on (correlation ids, flow-control flags, outcome). Pass `--full` to restore the pre-0.5.3 verbose body.

```bash
# default (concise)
codex-team -b $TOK status
# {"ok":true,"data":{"token":"...","live_sessions":2,"retained_events":7,"retained_limit":100,"pending_requests":1,"app_server_count":3}}

# --full
codex-team -b $TOK status --full
# {"ok":true,"data":{"token":"...","created_at":"...","last_active_at":"...","live_sessions":2,...,"daemon":{"pid":...,"started_at":"...","data_dir":"..."}}}

# --short (plain text — unchanged from 0.5.2)
codex-team -b $TOK status --short
# user=... live=2 pending=1 retained=7/100 app_servers=3 daemon_age=1m
```

Byte-size observed end-to-end: `status` 2.6×, `session info` 2.5×, `daemon config list` ~3×, `session context` ~6× smaller by default. `--short` and `--full` are mutually exclusive. Errors are never projected — error envelopes always surface in full.

## New: `codex-team doctor`

Eight ordered environment checks, plain-text output, exit `0` / `1` / `2`:

```
[OK] node=24.13.0
[OK] codex=codex-cli 0.122.0
[WARN] codex-team not on PATH; use <plugin-root>/bin/codex-team
[OK] data_dir=/home/wzq/.codex-team writable
[OK] socket_bind permitted
[OK] daemon running, pid=915313
[OK] daemon_socket reachable
[OK] dist current
=== HEALTHY ===
```

Run it the first time anything looks wrong — it catches the class of "silent daemon hang" that dogfood v1 was 1/10 and 2/10 over. `--short` gives a one-line grep-friendly summary.

## New: cohort operations

Stop hand-rolling shell loops for multi-session dispatch.

```bash
# broadcast one prompt to N live sessions in parallel
codex-team -b $TOK message send-many audit lint typecheck "run pending checks"
# returns {results:[{session, turn_id, started, queue_id, queued_depth} | {session, ok:false, error}]}

# block on ALL (or ANY)
codex-team -b $TOK message wait --all audit lint typecheck --timeout 300
codex-team -b $TOK message wait --any audit lint typecheck --timeout 60

# batch detach by glob
codex-team -b $TOK session detach --all --match 'mapper-*'
codex-team -b $TOK session detach --all --graceful
```

Exit codes reflect partial failure: `message send-many` exits `1` if any target failed; `wait --all` exits `0` / `1` / `124` based on overall outcome.

## New: detached-thread lifecycle

Codex's app-server exposes these; 0.5.3 finally surfaces them.

```bash
codex-team -b $TOK session archive <name|thread_id> [--and-detach]
codex-team -b $TOK session unarchive <thread_id>
codex-team -b $TOK session rename <name|thread_id> <new-name> --detached-ok
codex-team -b $TOK session rollback <name|thread_id> --to-turn <turn_id> [--detach-after]
```

Rollback forks from the target turn, hard-detaches the source, archives the old thread as `<name>-pre-rollback-<ISO8601>`, and rebinds the session name to the new fork.

## New: fleet observability

```bash
# one snapshot of every live session for your token
codex-team -b $TOK session health --all
codex-team -b $TOK session health --all --only-unhealthy
codex-team -b $TOK session health --all --state crashed,closed

# cross-user daemon view (no -b required)
codex-team daemon fleet status
codex-team daemon fleet status --users claude-alice,claude-bob

# retrospective per-session event viewer (since 0.5.2 `turn.completed` is compact)
codex-team -b $TOK session events <name> --limit 100
codex-team -b $TOK session events <name> --type turn.completed,item.completed
codex-team -b $TOK session events <name> --turn <turn_id> --summary
codex-team -b $TOK session events <name> --follow
codex-team -b $TOK session events <name> --by-tool    # item.completed tally by type
```

## New: `session list --all` actually browsable

Flags exposed: `--cursor`, `--limit`, `--archived only|exclude|include`, `--state <csv>`, `--owner self|any|<token>`, `--loaded-only`. Pagination works.

## New: skill-bundled profile library

The canonical role profiles (`fixer`, `reviewer`, `planner`, `tester`, `explorer`) now live in the skill itself (`skills/configure-codex-team/profiles-library.md`) as explicit `session new` flag bundles. **No more `--profile` silently defaulting because `~/.codex/config.toml` was missing** — the library is in the agent's skill context. User-local `~/.codex/config.toml` profiles are still supported for custom tuning via `--profile <name>`.

Playbooks (`skills/codex-team-playbooks/*.md`) now expand the flag bundle inline instead of relying on `--profile <name>`.

## Bug fixes

- **`turn.error` wedge** (R1 F1): `turn.error` with `will_retry=false` now clears `queue.currentTurnId` and drains the next queued item. Previously the queue got stuck forever.
- **Teardown tombstone race** (R1 F2): split `TurnQueues.dispose()` into `markTeardown` + `finalDispose`. Late `server_request` traffic from a dying app-server during detach/close/seize is now rejected with `-32000 session torn down` instead of creating phantom pending entries.
- **Rename didn't rekey queue/pending** (R1 H1): `session rename` now migrates queue state and pending-request `session_name`.
- **Failed queued dispatch broke FIFO** (R1 H2): a failed queued head stays at the head; newer sends queue behind it. After `retry.max_attempts` failures, emit `turn.queued_dropped` and resume.
- **`monitor events --cursor` could persist synthetic ids** (R1 H3): synthetic `monitor-overflow-N` chunks are now tagged `ackable: false`; CLI skips ack-scheduling; daemon rejects non-`evt-N` ids as a safety net.
- **Compaction double-write** (R1 H4): event-log compaction now snapshots and drains pending writes under the per-user lock, so later appends can't appear twice.
- **Cursor stale-lock reclaim not exclusive** (R1 H5): reclaim now unlinks the stale lock, reacquires with `open("wx")`, and verifies ownership via pid/nonce.

## Performance

- **Event ring buffer is O(1)** (R2 F1): replaced `Array.shift()` with a proper circular buffer + `event_id → slot` index. Append, `listSince()`, and `oldestId()` are all O(1).
- **Overflow-flush starvation fixed** (R2 F2): timer uses monotonic earliest-deadline; subsequent appends can't push a pending flush out indefinitely. Hard backlog cap emits `warning` event `kind: "event_log_backlog_overflow"` when exceeded.
- **`sessions.json` no longer rewritten per event** (R2 H1): volatile runtime fields (`current_turn_id`, `items_in_turn`, `pending_approvals`, token usage) are in-memory only. Coarse fields persist with a small `session.persist_debounce_ms` (default 50 ms) debounce.
- **Cursor auto-update coalesced** (R2 H2): in-memory cursor updates on every ack; disk rewrite debounced by `monitor.cursor_persist_debounce_ms` (default 200 ms, hot). Stream close and daemon shutdown force-flush.

## Error surfacing

- **Stale `daemon.pid` + `daemon.sock`**: detected at startup; CLI returns `daemon_unreachable` with `data.pid`/`pid_path`/`sock_path` and a manual cleanup hint instead of hanging.
- **`listen()` EPERM** is translated to `socket_bind_denied` with `data.suggested_action: "run codex-team doctor for details"`.
- **Pre-logger daemon bootstrap failures** write one-line `[codex-team-daemon-bootstrap] ...` JSON to `daemon-spawn.stderr`; CLI tails it and attaches as `data.bootstrap_stderr`.
- **Missing profile** is no longer silent — skill-library profiles are always available; user-local `--profile <name>` misses still return the app-server's error.

## Breaking changes

- **Default JSON response body is narrower.** Scripts that rely on e.g. `resp.data.thread_id` from `message send` should pass `--full` OR read from the narrower concise shape. Every concise projection is documented in `src/format/compact.ts` and covered by `tests/compact-format.test.ts`.

## Migration

```bash
# update package + rebuild
cd plugins/codex-team
npm install
npm run build

# restart daemon to pick up the new code
codex-team daemon restart

# verify
codex-team version                 # cli_version: 0.5.3
codex-team doctor                  # first-time sanity check
codex-team daemon status --full    # daemon_version: 0.5.3 after restart
```

If you have scripts that depended on pre-0.5.3 response fields, the fastest migration is `--full`; long-term, read the compact shape (fewer bytes, fewer surprises).
