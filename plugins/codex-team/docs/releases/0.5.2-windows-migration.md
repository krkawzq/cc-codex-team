# codex-team 0.5.2 Windows migration note

> For the full release context, CLI/event changes, and upgrade steps, start with [0.5.2 release notes](./0.5.2.md).

Pre-0.5.1 Windows installs could store state under `%HOME%\\.codex-team`. Starting in 0.5.1, the default root follows `os.homedir()`, which normally resolves to `%USERPROFILE%\\.codex-team`.

If 0.5.2 detects the old `%HOME%` path still exists while the new default path is empty, it emits a warning and leaves the files untouched. Move the directory manually if you want to keep the existing daemon state, users, sessions, and event logs.

Example:

- legacy path: `%HOME%\\.codex-team`
- new default: `%USERPROFILE%\\.codex-team`

No automatic migration is performed.
