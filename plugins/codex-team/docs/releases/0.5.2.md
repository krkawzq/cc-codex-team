# codex-team 0.5.2

Date: `2026-04-23`  
Tag: `v0.5.2`

0.5.2 is an operational release: better live-session recovery, better event-stream ergonomics, and a cleaner CLI surface for orchestration. The headline changes are `session health`, `session heal`, `message wait`, named cursors, summary event streaming, and a deliberately compact `turn.completed` payload.

## TL;DR

- You can now inspect and repair dead live sessions directly with `session health` and `session heal`.
- `message wait` replaces hand-rolled polling loops, and `monitor events --summary --cursor <name>` makes high-fanout orchestration resumable.
- Event consumers must migrate off embedded `turn.completed` items; detailed turn content now comes from `message history` / `message tail`.

## New CLI surface

- `session health <name>`: live snapshot with `busy`, `current_turn_id`, pending request counts, `token_usage_last_turn`, `app_server_alive`, and more.
- `session heal <name> [--force]`: re-attach a crashed or dead live session to a fresh app-server; `--force` clears half-baked runtime state first.
- `message wait <name> [--for <turn_id>] [--timeout <s>]`: block for turn completion with exit codes `0` / `1` / `124`.
- `cursor save|get|list|delete`: daemon-maintained named cursors for resumable event consumption.
- `monitor events --summary`: compact NDJSON `{id, ts, type, session, key}` for orchestration dashboards and Monitor panes.
- `monitor events --cursor <name>`: resume from a named cursor and auto-advance it after delivery; mutually exclusive with `--since`.
- `session new --auto-approve "<patterns>"`: per-session auto-approve rules using CSV and `/regex/flags`.
- `message approval ... --kind <kind>`: optional CLI-side validation hint for approval shortcuts.
- `--short` expanded to `status`, `daemon status`, `daemon user list`, `session info`, `session list`, and `message history`.
- `message history --truncate <bytes>` and `message tail --truncate <bytes>`: clip rendered bodies without changing the inline/block threshold.
- `daemon status` now includes `session_count`, `dist_built_at`, `dist_age_seconds`, and `source_newer_than_dist`.

## Breaking changes

### `turn.completed` is now compact

`turn.completed` no longer embeds the turn object or items array. The payload is now:

```json
{
  "turn_id": "...",
  "status": "completed | errored | cancelled",
  "duration_ms": 1234,
  "items_count": 5,
  "token_usage": { "...": "..." },
  "ended_at": 1712345678901,
  "turn_items_included": false
}
```

Migration notes:

- Stop reading embedded items from `turn.completed.payload.turn` or `turn.completed.payload.items`.
- Switch from `item_count` to `items_count`.
- Use `message history` or `message tail` when you need assistant messages, shell output, diffs, tool results, or reasoning text.

### Windows legacy `%HOME%` data-dir detection

0.5.2 warns when it detects an older `%HOME%\.codex-team` layout and the newer `os.homedir()` location is still empty. See [0.5.2-windows-migration.md](./0.5.2-windows-migration.md).

## New events

- `auto_approved`
- `session.crashed`
- `session.closed`
- `approval.request_cancelled`
- `user_input.request_cancelled`
- `session.pending_dropped`
- `event_log_header` (internal schema metadata in the persisted event log)

## Reliability fixes

- Daemon startup now reconciles stale persisted live sessions and marks them `crashed` instead of pretending they are still healthy.
- Restart-time pending loss is surfaced explicitly with `session.pending_dropped`.
- Live teardown paths and restart reconciliation now emit symmetric approval/user-input cancellation events.
- Auto-approve matching tolerates throwing matchers: bad patterns are logged and treated as non-matches.
- Cursor persistence is now cross-process safe with atomic writes and stale-lock recovery.
- `message approval` / `message answer` no longer block on warning-event persistence.
- Daemon spawn retry captures stderr to a stable path for startup debugging.
- Windows legacy `$HOME` probing now emits a one-shot migration warning instead of silently splitting state.

## Developer-facing

- Runtime version reporting is now SSOT-backed and verified across source, manifests, and built `dist/main.js`.
- `scripts/bump-version.mjs` replaces the older shell-only bump helper and updates both `package.json` and `.claude-plugin/plugin.json`.
- Markdown renderer coverage now includes snapshot fixtures under `tests/fixtures/markdown/*` and `tests/markdown-snapshot.test.ts`.
- Renderer additions include dedicated output for `mcpToolCall`, `hook.*`, `autoApprovalReview`, and long-form `reasoning`.

## Upgrading

Recommended:

```bash
cd plugins/codex-team
npm run bump-version 0.5.2
```

Manual path:

1. Update `package.json` to `0.5.2`.
2. Update `.claude-plugin/plugin.json` to `0.5.2`.
3. Run `npm run build`.
4. Verify `node dist/main.js version`.
5. Run `npm test && npm run typecheck`.
